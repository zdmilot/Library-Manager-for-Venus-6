Search & Filtering
Venus Library Manager » GUI Overview » Search & Filtering

Overview

The Venus Library Manager search bar provides a powerful inline filtering system that goes beyond simple text queries. You can combine free-text search with structured tag chips (#tag) and author chips (@author) to precisely narrow your library results. All filter types can be mixed freely in a single query and are applied together.

The search bar is located at the top of the main window. Press Ctrl+F from anywhere in the application to focus it instantly. When a search query is active, the UI enters search mode, temporarily suspending the active navigation tab and displaying consolidated results. Clearing the search restores the previously active tab.

Basic Text Search

Type any text directly into the search bar to perform a case-insensitive search across all library metadata:

  • Library name
  • Author and organization
  • Description
  • Tags
  • Public function names parsed from .hsl files (including system libraries)

Results update live as you type. Text tokens appear inline in the search bar alongside any tag or author chips, flowing together as a continuous stream.

Tag Chips (#tag)

Tag chips let you filter libraries by their assigned tags. A tag chip appears as a small colored pill with a # prefix displayed inside the search bar.

Creating a Tag Chip

  1. Type # followed by the tag name (for example: #pipetting).
  2. Press Space, Enter, or Tab to commit the tag. The text is converted into an inline chip.
  3. You can also click a tag badge directly on any library card to instantly create a #tag chip in the search bar.

Tags do not allow spaces. Typing a space after the tag name is the quickest way to commit it.

Editing a Tag Chip

Click the label text of an existing tag chip. The chip transforms into an editable input field pre-filled with the current value. Edit the text and press Space, Enter, or Tab to confirm, or press Escape to cancel. If you clear the chip text completely and confirm, the chip is removed.

Removing a Tag Chip

  • Click the × button on the right side of the chip to remove it immediately.
  • Backspace twice: When the text cursor is at the beginning of the input (position 0), press Backspace once to highlight the preceding chip in red. Press Backspace a second time to delete it. Moving the cursor or clicking elsewhere cancels the pending deletion.
  • Delete key: When editing an inline text token and the cursor is at the end of the text, press Delete once to highlight the next chip in red. Press Delete again to remove it.

Tag Matching Behavior

When one or more tag chips are active, only libraries that match all specified tags are shown (AND logic). Tag matching is case-insensitive and uses substring matching on library tags. System libraries include virtual system tags (for example: #system), so tag filters work across both user-installed and system libraries.

Author Chips (@author)

Author chips let you filter libraries by author or organization name. An author chip appears as a small colored pill with an @ prefix and a person icon, displayed inside the search bar.

Creating an Author Chip

  1. Type @ followed by the author or organization name (for example: @Hamilton Company).
  2. Press Tab or Enter to commit the author chip.

Note: Unlike tag chips, author chips allow spaces within the name (for example: @Hamilton Company). Because of this, the Space key does not commit author chips — only Tab or Enter will. However, the application tracks how many spaces are used in known author names from the publisher registry, and if you type more spaces than any known author contains, the chip is automatically committed for convenience.

Editing an Author Chip

Click the label text of an existing author chip. The chip transforms into an editable input field pre-filled with the current value. Edit the text and press Tab or Enter to confirm, or press Escape to cancel. If you clear the text completely and confirm, the chip is removed.

Removing an Author Chip

Author chips are removed the same way as tag chips:

  • Click the × button on the chip.
  • Use the Backspace twice pattern from the main input.
  • Use the Delete twice pattern from an inline text editor.

Author Matching Behavior

When one or more author chips are active, a library must match at least one of the specified authors to be shown (OR logic among author filters). The match checks the library's author field, or its organization field when the chip icon displays a building/organization symbol. Matching is case-insensitive and uses substring matching.

When author chips are combined with tag chips and text, all filter types must be satisfied together: the library must match all tag chips AND at least one author chip AND the text query.

Inline Text Editing

When you have a mix of text, tag chips, and author chips in the search bar, you may need to edit the text portions that are interspersed between chips. Instead of clearing the entire search, you can edit text tokens in place:

  1. Click on any text token displayed between chips. The text turns into an inline editor input sized to match the text width.
  2. Edit the text as needed.
  3. Press Enter or Tab to confirm the edit, or press Escape to cancel and revert.
  4. The surrounding chips remain in their positions — only the clicked text segment is modified.

If you delete all text from an inline text editor and confirm, the empty text token is automatically cleaned up.

Autocomplete Suggestions

When you begin typing a # or @ prefix in the search bar, an autocomplete dropdown appears below the search input offering suggestions from known tags and authors.

How Autocomplete Works

  • Tag autocomplete: Type # followed by one or more characters. The dropdown shows all known tags that contain your typed text, sorted alphabetically. Each suggestion shows a # icon.
  • Author autocomplete: Type @ followed by one or more characters. The dropdown shows all known authors and organizations that contain your typed text, sorted alphabetically. Author entries show a person icon; organization entries show a building icon.
  • Suggestions are drawn from the publisher registry, which is automatically populated from all imported library metadata (see Publisher Registry below).
  • The matching text within each suggestion is highlighted in bold so you can quickly identify the match.

Using the Autocomplete Dropdown

ActionDescription
Arrow Down / Arrow UpNavigate through the suggestions list. The selected item is visually highlighted.
Enter or TabAccept the currently highlighted suggestion and insert it as a chip.
EscapeClose the dropdown without selecting anything.
ClickClick any suggestion to select and commit it immediately.

The dropdown closes automatically when the input loses focus, when the typed text no longer starts with # or @, or when there are no matching results.

Tip: Autocomplete is the fastest way to search for authors with multi-word names. Type @Ham and select “Hamilton Company” from the dropdown instead of typing the full name.

Publisher Registry

The publisher registry is a local index that tracks all known authors, organizations, and tags encountered across all imported libraries. It is stored as publisher_registry.json in the application's user data directory.

How the Registry Is Built

  • Every time a library is imported (single package or archive), the library's author, organization, and tags fields are automatically registered.
  • The registry is also updated when libraries are saved, exported, or registered from unsigned sources.
  • The registry can be fully rebuilt from all currently installed libraries at any time (this happens automatically during startup if the registry file is missing or corrupt).

What the Registry Stores

FieldDescription
publishersAn array of unique author names that have appeared in library metadata.
organizationsAn array of unique organization names that have appeared in library metadata.
tagsAn array of unique tags that have appeared in library metadata.

The registry is used to power autocomplete suggestions and to determine the maximum number of spaces any known author name contains (used for the auto-commit heuristic described in the Author Chips section above).

Search Token Order

The search bar preserves the exact order in which you entered your tokens. If you type pump, then add #liquid-handling, then add @Hamilton, the display query shown in the header reflects that exact sequence. Tokens are never reordered or grouped by type.

Search Mode Behavior

When any search query is active (text, tags, or authors), the application enters search mode:

  • The currently active navigation tab is temporarily deselected.
  • All matching libraries are displayed in a flat, consolidated list regardless of grouping.
  • The active display query is shown in the search bar with all chips and text tokens rendered inline.
  • Clearing the search bar (removing all chips and text) exits search mode and restores the previously active tab.

Combined Filtering Logic

When multiple filter types are active simultaneously, they are combined as follows:

Filter CombinationLogic
Multiple #tag chipsAND — Library must match all specified tags.
Multiple @author chipsOR — Library must match at least one specified author or organization.
Tags + AuthorsAND — Library must satisfy all tag filters AND at least one author filter.
Tags + Authors + TextAND — Library must satisfy all tag filters, at least one author filter, and contain the text query in its searchable fields.

Quick-Start Examples

QueryWhat It Does
pipetteFinds all libraries whose name, author, description, tags, or function names contain “pipette”.
#liquid-handlingFinds all libraries tagged with “liquid-handling”.
@HamiltonFinds all libraries authored by “Hamilton” (or with “Hamilton” as the organization).
#pipetting @Hamilton pumpFinds libraries tagged “pipetting”, authored by “Hamilton”, and containing “pump” in any searchable field.
#systemShows Hamilton system libraries (they carry the virtual system tag).

Note: Typing a bare # or @ followed immediately by a space does not create a chip — the prefix character is silently removed and no empty chip is created.

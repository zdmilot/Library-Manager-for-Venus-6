Overview

Library Manager includes a .NET COM object (VenusLibraryManager.dll) that enables integration with Hamilton VENUS HSL methods, VBScript, and any COM-capable language. The COM object provides a superset of the CLI commands, including additional convenience methods (e.g., GetLibrary, GetAuditTrail, GetSettings, extended Ex variants with full parameter control).

32-bit Only: The COM DLL is compiled as x86 (32-bit) and must be registered using the 32-bit RegAsm.exe. Hamilton VENUS is a 32-bit application — using the 64-bit RegAsm will make the COM object invisible to VENUS, causing "Class not registered" errors.

Registration

Automatic (Installer)

The installer automatically registers the COM DLL during installation and unregisters it during uninstallation. No manual action is required.

Manual Registration

REM Register (requires Administrator)
C:\Windows\Microsoft.NET\Framework\v4.0.30319\RegAsm.exe /codebase "C:\Program Files (x86)\Library Manager\com\VenusLibraryManager.dll"

REM Unregister
C:\Windows\Microsoft.NET\Framework\v4.0.30319\RegAsm.exe /unregister "C:\Program Files (x86)\Library Manager\com\VenusLibraryManager.dll"

Do NOT use Framework64. Always use C:\Windows\Microsoft.NET\Framework\v4.0.30319\RegAsm.exe (the 32-bit version).

Verify Registration

Run the included verification script:

"C:\Program Files (x86)\Library Manager\com\verify-com.bat"

Architecture

The COM object communicates with the application’s service layer via a lightweight bridge process (com-bridge.js). When you call a COM method, it:

  1. Spawns com-bridge.js with the command name and JSON arguments
  2. Waits for the bridge to execute the operation via the shared service layer
  3. Returns the JSON response string to the caller

This architecture ensures both interfaces (CLI and COM) share identical business logic and security enforcement.

Return Format

All methods return JSON strings. Successful operations return:

{"success":true,"data":{...}}

Failed operations return:

{"success":false,"error":"Description of error"}

Usage Examples

VBScript

' Create the COM object
Set mgr = CreateObject("VenusLibraryManager.LibraryManager")

' List all installed libraries
WScript.Echo mgr.ListLibraries()

' Import a library package
result = mgr.ImportLibrary("C:\Packages\MyLibrary.hxlibpkg")
WScript.Echo result

' Delete a library (soft delete)
result = mgr.DeleteLibrary("MyLibrary")
WScript.Echo result

' Check the last error
WScript.Echo "Last error: " & mgr.LastError

' Clean up
Set mgr = Nothing

Hamilton VENUS HSL

// Declare the COM object variable
variable libMgr;
variable result;

// Create the Library Manager COM object
libMgr = CreateObject("VenusLibraryManager.LibraryManager");

// List installed libraries
result = libMgr.ListLibraries();
Trace(result);

// Import a package
result = libMgr.ImportLibrary("C:\\Packages\\MyLib.hxlibpkg");
Trace(result);

// Verify a package before import
result = libMgr.VerifyPackage("C:\\Packages\\MyLib.hxlibpkg");
Trace(result);

C# (.NET)

// Late binding via COM ProgID
Type t = Type.GetTypeFromProgID("VenusLibraryManager.LibraryManager");
dynamic mgr = Activator.CreateInstance(t);

// List libraries
string json = mgr.ListLibraries();
Console.WriteLine(json);

// Import a package with all options
string result = mgr.ImportLibraryEx(
    @"C:\Packages\MyLib.hxlibpkg",
    true,   // force overwrite
    false,  // noGroup
    false,  // noCache
    null    // authorPassword
);
Console.WriteLine(result);

PowerShell (via COM)

# Create the COM object
$mgr = New-Object -ComObject VenusLibraryManager.LibraryManager

# List libraries
$json = $mgr.ListLibraries()
$json | ConvertFrom-Json | Format-Table

# Import a package
$result = $mgr.ImportLibrary("C:\Packages\MyLib.hxlibpkg")
Write-Host $result

# Clean up
[System.Runtime.InteropServices.Marshal]::ReleaseComObject($mgr) | Out-Null

Events

The COM object fires the following events for clients that support COM event sinking:

EventParametersDescription
OperationCompletedoperation, success, resultJsonFired after each operation completes
ErrorOccurredoperation, errorMessageFired when an error occurs

Error Handling

Check the LastError property after any operation, or parse the JSON result for the "success" field:

result = mgr.ImportLibrary("C:\bad\path.hxlibpkg")
' Parse result JSON to check success
If InStr(result, """success"":false") > 0 Then
    WScript.Echo "Error: " & mgr.LastError
End If

Troubleshooting

ProblemSolution
"Class not registered" errorRegister with 32-bit RegAsm.exe as Administrator. Never use Framework64.
"Cannot find node.exe"Install Node.js, or ensure the NW.js runtime is in the application directory.
Operations timeoutLarge imports/exports may take time. The default timeout is 5 minutes per operation.

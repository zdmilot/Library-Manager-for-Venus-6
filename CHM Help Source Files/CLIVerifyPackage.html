CLI: verify-package
Library Manager for Venus 6 » CLI Reference » verify-package

Description

Verifies the integrity of a .hxlibpkg or .hxlibarch file without importing it. This command validates both the outer binary container (HMAC-SHA256 over the entire file) and the inner signature.json (per-file integrity hashes). This is useful for validating packages before distribution, after file transfer, or in CI/CD pipelines.

Usage

node cli.js verify-package --file <path> [--json]

Options

FlagRequiredDescription
--file <path>RequiredPath to the .hxlibpkg file to verify
--jsonNoPrint machine-readable JSON output instead of human-readable text

Verification Process

  1. The .hxlibpkg file's binary container is validated: magic bytes, HMAC-SHA256 and payload length are checked. If the container is corrupted or tampered with, verification fails immediately.
  2. The container is descrambled to recover the internal ZIP archive
  3. The tool looks for a signature.json entry inside the internal ZIP
  4. If no signature.json is found, the package is reported as UNSIGNED (exit code 0)
  5. If signature.json is present:
    • SHA-256 hashes are recomputed for every non-signature entry in the ZIP
    • Each recomputed hash is compared against the stored hash in signature.json
    • The HMAC-SHA256 is recomputed from the file hash map and compared against the stored HMAC
  6. If all hashes and the HMAC match, the result is VERIFIED (exit code 0)
  7. If any hash or the HMAC does not match, the result is FAILED (exit code 1)

Possible Outcomes

ResultExit CodeMeaning
VERIFIED0Signature is present and valid - all file hashes match and the HMAC is correct
UNSIGNED0No signature.json found - the package is a legacy or third-party package created before signing was implemented
FAILED1One or more file hashes do not match, or the HMAC is invalid - the package may be corrupt or tampered with

Examples

:: Verify a package before importing
node cli.js verify-package --file MyLibrary.hxlibpkg

:: Machine-readable output for CI/CD pipelines
node cli.js verify-package --file MyLibrary.hxlibpkg --json

:: Verify a package after downloading from a network share
node cli.js verify-package --file "\\server\share\packages\LibraryA.hxlibpkg"

JSON Output Format

When --json is specified, the output is a JSON object:

{
  "file": "MyLibrary.hxlibpkg",
  "status": "VERIFIED",
  "algorithm": "HMAC-SHA256",
  "fileCount": 5,
  "created": "2026-01-15T10:30:00.000Z"
}

Signature Format

The signature.json entry inside a signed .hxlibpkg contains:

FieldTypeDescription
algorithmstringAlways "HMAC-SHA256"
createdstringISO 8601 timestamp of when the signature was generated
filesobjectMap of { "entryName": "sha256hex", ... } for every entry in the ZIP except signature.json itself
hmacstringHMAC-SHA256 computed over the canonical JSON representation of the files map

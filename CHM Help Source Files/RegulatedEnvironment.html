Regulated Environment Mode
Library Manager » Regulated Environment Mode

Overview

Regulated Environment Mode provides Windows security group-based access control for protected library management actions. When enabled, only users who belong to authorized Windows security groups can perform actions that modify the library installation — importing, deleting, and rolling back libraries. This ensures that library changes on shared instruments are restricted to authorized personnel.

This feature is designed for laboratories that require controlled access to library management operations. It leverages existing Windows security group infrastructure — no additional user management is needed beyond standard Active Directory or local group assignments.

Administrator Super Whitelist

Windows Administrators (members of the BUILTIN\Administrators group) are on a super whitelist and receive full access to all library management actions and all settings, regardless of whether they are assigned to any Hamilton-specific security group. This ensures that system administrators always retain complete control over the application.

When an Administrator opens the Settings dialog, an unlock icon is displayed next to each settings section header, providing a visual indicator that the user has elevated access as an Administrator.

How It Works

When regulated environment mode is enabled, every protected action triggers a Windows group membership check before proceeding:

  1. The system runs whoami /groups /fo csv /nh to retrieve the current user’s Windows security group memberships
  2. Group results are cached for 15 seconds to avoid repeated subprocess calls during rapid operations
  3. The user’s groups are compared against two lists:
    • Super whitelist — Windows Administrators are always allowed (highest priority)
    • Allow list — users in these groups are permitted to perform the action
    • Deny list — users in these groups are explicitly blocked
  4. If the user is not in any recognized group, access is denied by default (deny-by-default policy)

Group name matching is case-insensitive. The system checks whether any part of the full Windows group path contains the group name (e.g., DOMAIN\Lab Method Programmer matches Lab Method Programmer).

Deny List Takes Priority

If a user belongs to both an allow group and a deny group, the deny list takes priority and access is blocked. This follows a conservative security model where explicit denials always override grants. However, Windows Administrators bypass this check entirely as part of the super whitelist.

Toggling Regulated Environment Mode

Enabling the Mode

  1. Open the Settings dialog (click the overflow menu → Settings, or press the gear icon)
  2. Locate the Regulated Environment section
  3. Check the Enable regulated environment mode checkbox
  4. A confirmation dialog will appear explaining that enabling this mode will restrict library actions to authorized users only
  5. Confirm to apply the change

Disabling the Mode

  1. Open the Settings dialog
  2. Uncheck the Enable regulated environment mode checkbox
  3. A confirmation dialog will appear warning that disabling the mode will allow all users to perform library actions
  4. Confirm to apply the change

Who Can Toggle the Setting

Only users who are themselves authorized (members of an allow-listed Windows security group or Windows Administrators) can enable or disable regulated environment mode. If an unauthorized user attempts to change the toggle, the action is blocked and the checkbox is reverted to its previous state.

In an unregulated installation, any user regardless of group membership can change all application settings except toggling regulated environment mode on. Only members of authorized groups (Lab Method Programmer, Lab Service) or Windows Administrators can enable regulated mode. This prevents accidental or false enabling of this mode by unauthorized users.

When regulated environment mode is active, the Settings dialog shows a status indicator beneath the checkbox:

  • Locked — the current user is not authorized to change this setting (the checkbox is disabled)
  • Authorized — the current user can change this setting
  • Administrator — the current user has access as a Windows Administrator (super whitelist)

Protected Actions

When regulated environment mode is enabled, the following actions require authorization:

Rollback LibraryReverting a library to a previous versionLibrary detail modal → Rollback

Actions that do not modify the library installation (browsing, searching, exporting, viewing details, creating packages) are always available to all users regardless of the mode setting.

Unsigned Libraries in Regulated Mode

When regulated environment mode is enabled, the unsigned libraries feature is automatically disabled and cannot be re-enabled. This is because regulated environments require all packages to be signed — allowing unsigned library scanning would undermine the purpose of regulated mode.

Specifically, when regulated mode is turned on:

  • The Include existing unsigned libraries on the front page checkbox is unchecked and disabled
  • The Scan for new unsigned libraries each launch checkbox is unchecked and disabled
  • The Scan Now button is disabled
  • A warning message is displayed: “Unsigned libraries cannot be enabled in regulated environment mode. All packages must be signed.”
  • The Unsigned navigation group is hidden

If unsigned libraries were previously enabled and tracked, they remain in the database but are no longer visible or scannable while regulated mode is active. Disabling regulated mode restores the ability to toggle unsigned library scanning.

Default setting: Unsigned library scanning is off by default in all installations. It must be explicitly enabled in Settings. This ensures that newly deployed instruments do not automatically surface unmanaged files.

GitHub Links in Regulated Mode

When regulated environment mode is enabled, the Show GitHub repository links setting is automatically disabled and cannot be re-enabled. External links to source code repositories are not permitted in a regulated environment. This prevents users from navigating to external websites from within the application.

Specifically, when regulated mode is turned on:

  • The Show GitHub repository links in library details checkbox is unchecked and disabled
  • A warning message is displayed: “GitHub links cannot be enabled in regulated environment mode.”
  • GitHub repository URLs are hidden in library detail modals and import previews, even if the package contains a repository URL

Disabling regulated mode restores the ability to toggle GitHub links on.

Access Denied Dialog

When a user attempts a protected action without authorization, an Access Denied modal is displayed showing:

  • The name of the action that was attempted
  • The reason for denial (e.g., “You are in a deny-listed group” or “You are not in any authorized group”)
  • The current Windows username
  • The user’s detected Windows security group memberships

This information can be used by IT administrators to troubleshoot group assignment issues.

Authorization Decision Flow

Recommendations

  • Use Active Directory groups — Assign users to the appropriate security groups via Active Directory for centralized management across multiple instruments
  • Test group assignments — Before enabling regulated mode, verify that key users (e.g., method developers) are correctly assigned to Lab Method Programmer or Lab Service groups
  • Review the Access Denied dialog — If a user reports denied access, check the detected groups listed in the dialog to confirm their group membership is correctly configured
  • Combine with audit trail enforcement — For maximum traceability, enable both regulated environment mode and the audit trail comment/signature requirements in Settings

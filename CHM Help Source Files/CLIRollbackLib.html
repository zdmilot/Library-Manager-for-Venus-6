CLI: rollback-lib
Library Manager » CLI Reference » rollback-lib

Description

Reinstalls a previously cached version of a library from the package store. This effectively reverts a library to an earlier version by re-importing the cached .hxlibpkg, overwriting the current installation.

Usage

node cli.js rollback-lib --name <library-name> [--version <ver> | --index <n>]

Options

FlagRequiredDescription
--name <name>RequiredLibrary name to roll back
--version <ver>OptionalTarget version string to roll back to (e.g., "1.0.0")
--index <n>Optional1-based index from list-versions output
--lib-dir <path>OptionalOverride library installation directory
--met-dir <path>OptionalOverride methods installation directory
--db-path <path>OptionalOverride user data directory
--store-dir <path>OptionalOverride the package store directory where cached versions are stored
--no-groupOptionalDo not automatically assign the library to a group
--author-password <pw>OptionalRequired when rolling back a package published by a restricted author (e.g. Hamilton). See Regulated Environment.

Version Selection

You must specify either --version or --index. If neither is provided, the command prints the available cached versions and exits, prompting you to choose.

  • --version - Matches by the version string in the cached manifest. If multiple caches of the same version exist, the newest one is used.
  • --index - Uses the 1-based index from list-versions output to select a specific cached entry.

Process Steps

  1. Validates the library name
  2. Looks up all cached packages for the given library name
  3. Selects the target version by --version or --index
  4. Opens the cached .hxlibpkg and verifies its cryptographic signature
  5. Reads the manifest.json; if the package author is restricted, requires --author-password
  6. Runs the standard import flow (same as import-lib):
    • Extracts library files to Library\<LibName>\
    • Extracts demo methods to Methods\Library Demo Methods\<LibName>\
    • Updates the database record with the manifest metadata
    • Generates integrity hashes for tracked file types
  7. Reports the number of extracted files and installation paths

COM DLL Note

Note: If the rolled-back version contains DLLs that require COM registration (com_register_dlls in the manifest), the CLI will print a reminder. COM registration is only performed automatically by the GUI (using the 32-bit RegAsm.exe); via the CLI you must run the 32-bit RegAsm.exe /codebase manually: 
C:\Windows\Microsoft.NET\Framework\v4.0.30319\RegAsm.exe /codebase <dll>
Do NOT use Framework64 — VENUS is 32-bit. See COM DLL Registration for details.

Examples

Roll back by version string

node cli.js rollback-lib --name "HSLAppsLib" --version "1.0.0"

Roll back by list-versions index

:: First, list cached versions
node cli.js list-versions --name "HSLAppsLib"

:: Then use the index
node cli.js rollback-lib --name "HSLAppsLib" --index 2

Interactive mode (no version specified)

node cli.js rollback-lib --name "HSLAppsLib"

Output:

Available cached versions for "HSLAppsLib":

  [1] v1.2.0  (cached 2024-06-01T14:22:11Z)
  [2] v1.1.0  (cached 2024-03-15T09:10:05Z)

Error: Specify which version to rollback to with --version <ver> or --index <n>

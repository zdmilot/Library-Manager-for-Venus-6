Importing from VENUS Package Files (.pkg)
Library Manager » Importing from VENUS Package Files

Overview

Hamilton VENUS software distributes libraries, labware definitions, configuration files, and method examples inside proprietary .pkg package files. These binary packages are used by the VENUS installer and update system to deploy content to the Hamilton system directories. Library Manager can open, explore, and selectively import library content from these .pkg files.

This feature is especially useful for:

  • Extracting individual libraries from large VENUS installation packages
  • Merging multiple HSL and help files from a .pkg into a single managed library entity (e.g., frameworks like HSL Framework that bundle multiple .hsl and .chm files)
  • Exploring package contents before deciding what to import
  • Bringing legacy VENUS libraries under Library Manager control

Note: This is different from importing .hxlibpkg files. VENUS .pkg files use a completely different binary format and do not contain Library Manager metadata. When importing from a .pkg, the user provides the library metadata (name, author, version, etc.) manually.

PKG File Format

Hamilton .pkg files use a binary container format with the following structure:

  • Header: 46 bytes beginning with the magic string HamPkg, containing the entry count, creation timestamp (Windows FILETIME), and VENUS version string
  • Entry Table: N × 36-byte entries, each with an entry ID, flags, data offset, and data size
  • Data Blocks: Zlib-compressed data for each entry (8-byte header + deflate stream)
  • Manifest: An HxPars-format entry (flags=0) mapping entry IDs to absolute installation paths
  • Trailer: $$key=value$$ pairs with package metadata (author, description, etc.)

GUI Workflow

Step 1: Select a Package File

  1. Click the overflow menu (three-dot button) in the toolbar
  2. Select Import HamPkg
  3. In the modal that appears, either:
    • Click the drop zone to browse for a .pkg file
    • Drag and drop a .pkg file onto the drop zone

Step 2: Explore and Select Files

After loading the package, the modal displays:

  • Package Information: File name, VENUS version, author, creation date, file count, and format version
  • Category Filters: Tabs to filter by All, Library, Help, Labware, Config, or Methods
  • File List: Every file in the package with its name, category, relative path, and size. Click a file to toggle its selection. Use All / None buttons to bulk-select or deselect.

Library (.hsl, .hs_, .sub) and help (.chm, .pdf) files are auto-selected by default. You can add or remove files from the selection as needed.

Step 3: Configure Library Metadata

Below the file list, fill in the library details:

FieldRequiredDescription
Library NameYesName for the library entity. Auto-detected from HSL file names when possible.
VersionYesVersion string (e.g., 1.0.0). Defaults to 1.0.0.
AuthorYesLibrary author (minimum 3 characters). Auto-filled from the package trailer if available.
OrganizationNoPublisher organization.
VENUS CompatibilityYesCompatible VENUS versions (e.g., 6.1+). Auto-filled from the package header.
TagsNoComma-separated tags for categorization.
DescriptionYesBrief description of the library.

Install Path: By default, files are installed to <Library Root>\<LibraryName>. Check the Install to root option to place files directly in the Library root directory (useful for libraries that VENUS expects at a specific path).

Step 4: Import

Click Import as Library to install the selected files. The import process:

  1. Creates the library directory (and demo methods directory if method files are selected)
  2. Writes the selected files from the package to their destination directories
  3. Computes SHA-256 integrity hashes for tracked file types
  4. Parses public function signatures from HSL files
  5. Creates a database record in the installed_libs collection with full metadata and package_lineage tracking the .pkg source
  6. Assigns the library to the appropriate group
  7. Records an audit trail entry

Merging Files into a Single Library

One of the most powerful uses of HamPkg import is creating a single library entity from multiple files spread across a package. For example, the HSL Framework ships as part of a VENUS installation package and contains multiple .hsl files and .chm help files. With this feature, you can:

  1. Open the VENUS .pkg file
  2. Filter to Library and Help categories
  3. Select all the .hsl and .chm files that belong to the framework
  4. Enter a library name like HSLFramework
  5. Import them as a single managed library

The resulting library appears as one entity in Library Manager, with all files tracked together for versioning, export, and integrity verification.

File Categories

Files within a .pkg are automatically categorized based on their original installation path and file extension:

CategoryDescriptionTypical Extensions
LibraryHSL source files and compiled libraries.hsl, .hs_, .sub
HelpHelp files and documentation.chm, .pdf, .htm, .html
LabwareLabware definitions and rack files.rck, .tml, .ctr
ConfigConfiguration and system files.cfg, .xml, .ini
MethodsMethod files and examples.med, .met, .hsl (in Methods path)

OEM Restrictions

When the author or organization name matches a restricted OEM keyword, the same protections apply as with regular library imports:

  • An OEM authorization password is required to proceed
  • The library is automatically assigned to the protected OEM group

Limitations

  • .pkg files do not contain Library Manager metadata, so the user must provide all library information manually
  • COM DLL registration is not supported for HamPkg imports — DLLs can be extracted but are not registered automatically
  • Code signing and package caching are not available for .pkg imports since they are not Library Manager packages
  • Labware and configuration files can be selected for import but are placed in the library directory, not in their original VENUS system locations

CLI: list-versions
Library Manager » CLI Reference » list-versions

Description

Lists all cached .hxlibpkg versions for a given library. Every time a library is imported (via GUI or CLI), the package file is automatically saved to the package store. This command queries that store and displays each cached version with its metadata.

Usage

node cli.js list-versions --name <library-name> [--json]

Options

FlagRequiredDescription
--name <name>RequiredLibrary name to look up in the package store
--json, -jOptionalOutput results as a JSON array
--store-dir <path>OptionalOverride the package store directory (global option)

Package Store Location

Cached packages are stored in:

C:\Program Files (x86)\Library Manager\local\packages\<LibraryName>\

Files follow the naming pattern: <Name>_v<Version>_<timestamp>.hxlibpkg

Output Fields

Each cached entry displays the following information:

FieldDescription
VersionLibrary version from the manifest
AuthorAuthor from the manifest (or - if not set)
CreatedOriginal creation date from the manifest (or - if not set)
CachedDate the package was stored in the cache
SizePackage file size in KB
FileFilename of the cached .hxlibpkg

Examples

Table Output (Default)

node cli.js list-versions --name "HSLAppsLib"

JSON Output

node cli.js list-versions --name "HSLAppsLib" --json

No Cached Versions

If no packages have been cached for the given library name, you will see:

No cached packages found for "HSLAppsLib".

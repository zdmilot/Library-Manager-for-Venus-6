Data Model
Library Manager for Venus 6 » Data Model

Overview

Library Manager for Venus 6 uses diskdb, a lightweight JSON-backed database, to persist all application state. The database resides in the db/ directory within the application folder. Each collection is stored as a separate .json file.

The db/ directory ships with the application and contains read-only reference data. The local data directory (local/ within the application’s own install folder) stores all mutable working data including installed library records, group assignments, the audit trail, and cached packages. This separation ensures that all data resides within the application directory and is globally accessible to every user account on the PC.

file_hashes Object

Maps each integrity-tracked file to its SHA-256 hash. Only files with extensions .hsl, .hs_, .sub, and COM .dll files are tracked.

"file_hashes": {
  "ExampleLib.hsl": "612b778dde75f7e93c2894845e1f9f7e...",
  "ExampleLib.hs_": "cdd4d99218e02df8972ab8dbb1b6be43...",
  "ExampleLib.dll": "f30cb612945c0af795b134cd6b964e70..."
}

public_functions Array

Each entry describes a public function parsed from the library's .hsl files:

{

  ],
  "returnType": "variable",
  "doc": "Function documentation from comments...",
  "file": "ExampleComHelper.hsl"

groups Collection

Defines the navigation groups (tabs) shown in the GUI. The application uses a two-tier group architecture:

Default Groups (Hardcoded)

Seven built-in groups are defined in a DEFAULT_GROUPS map in the application source code. These are never stored in groups.json - they are resolved in-memory at runtime. A startup migration automatically strips any stale default-group records from the JSON file to prevent duplicates.

IDNameIconSidePurpose
gAllAllfa-homeleftShows all installed libraries
gRecentRecentfa-historyleftRecently accessed libraries
gStarredStarredfa-starleftLibraries starred (favorited) by the user
gHamiltonHamiltonfa-check-circle (solid)leftProtected Hamilton-authored libraries
gFoldersImportfa-downloadrightImport workflow tab
gEditorsExportfa-uploadrightExport workflow tab
gHistoryHistoryfa-listrightAction history

The gHamilton group has an additional "protected": true property that prevents it from being deleted or renamed. See Library Grouping for full details on Hamilton group protections.

User-Created Groups (Stored in groups.json)

User-created groups have auto-generated _id values and "default": false. Each group has a name, FontAwesome icon class, and display metadata.

{
  "_id": "gAll",
  "name": "All",
  "icon-class": "fa-home",
  "default": true,
  "navbar": "left",
  "favorite": true
}

Default Groups

The following default groups are hardcoded and not stored in groups.json. See the table above for the complete list. The code sample below shows what a default group entry looks like in the DEFAULT_GROUPS map (but this is not persisted to disk):

// In-memory only - not stored in groups.json
{
  "_id": "gHamilton",
  "name": "Hamilton",
  "icon-class": "fa-check-circle",
  "default": true,
  "protected": true,
  "navbar": "left",
  "favorite": true
}

User-created groups have auto-generated _id values and "default": false.

Maps groups to their member libraries. Each record links a group-id to an array of library _id values.

settings Collection

Stores application preferences. Always a single record with _id of "0".

Storage note: Settings are stored in local/settings.json within the application’s install directory. The db/settings.json file in the program directory serves as a read-only default template that is copied to local/ on first run.

Local Data Directory

All mutable application data is stored in the local/ directory under the current user profile:

%LOCALAPPDATA%\Library Manager for Venus 6\local\

This directory contains the live, mutable copy of the database that reflects the current installation state. It also contains:

  • audit_trail.json - A persistent audit trail recording all library lifecycle events (package creation, import, delete, rollback) with environmental context such as the Windows username, Windows OS version, VENUS version, and hostname. See Library Audit Log for details.
  • packages/ - Cached .hxlibpkg package backups organized by library name, used for version rollback and integrity repair
  • exports/ - Default output directory for exported packages and archives

No mutable application database data is stored in ProgramData or the VENUS HAMILTON\Library directory. The VENUS Library directory (C:\Program Files (x86)\HAMILTON\Library) is used only for storing unpacked library files (HSL, DLL, etc.) and their associated resources.

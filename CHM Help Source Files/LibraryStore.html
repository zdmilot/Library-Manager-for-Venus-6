Library Store
Library Manager » Library Store

Overview

The Library Store provides a built-in marketplace for browsing, downloading, and installing community-curated library packages directly from within Library Manager. The store connects to a public GitHub repository that hosts a catalog of available libraries, and offers one-click installation with automatic dependency resolution.

The store is accessible from the Store button () in the navigation bar, located between the search and import buttons.

Opening the Store

Click the Store button in the navigation bar to open the Library Store modal. On first open, the application fetches the package catalog from GitHub over HTTPS. A loading spinner is shown while the catalog is being retrieved. Subsequent opens use the cached catalog unless you click Refresh.

Browsing Libraries

The store displays available libraries as cards in a grid layout. Each card shows:

  • Library icon (or a default book icon if no image is provided)
  • Library name and version
  • Author and organization
  • Description
  • Star ratings (community average and review count)
  • Tags (up to four displayed, with a count for additional tags)
  • Install status:
    • Download and Install button — library is not installed
    • Installed (green, disabled) — the same version is already installed
    • Update available badge — a newer version is available in the store

Update Badge

When the store catalog contains newer versions of libraries you already have installed, a red badge appears on the Store button in the navigation bar showing the count of available updates. Individual cards also show an orange update badge with the new version number.

Search

The store toolbar includes a search bar that filters the library grid in real time. The search matches against library names, authors, organizations, descriptions, and tags. A clear button (×) resets the search.

Sorting

Library Detail View

Click any library card to open a detailed view showing comprehensive information about the package:

  • Library icon and name
  • Version selector — a dropdown listing all available versions, with the latest marked as "(latest)"
  • Author and organization
  • VENUS compatibility and created date
  • Description
  • GitHub repository link (opens in external browser)
  • Package file link (opens in external browser, points to the GitHub repository)
  • Tags
  • Library files — full file list with destination install paths and COM registration badges
  • Demo method files — full file list with destination install paths
  • Help files — included .chm help files
  • Bin files — binary files to be installed to the Hamilton Bin directory
  • Dependencies — list of required libraries with install status (installed, available in store, or not in store)
  • Overwrite warning — shown if the library is already installed, noting version differences
  • Community reviews and ratings

Version Selector

The version dropdown in the detail view header lists all available versions of the library in the store. Selecting a different version refreshes the detail view with that version's metadata, file lists, and install paths. The latest version is always listed first and marked with "(latest)".

Installing from the Store

There are two ways to install a library from the store:

Quick Install (From Card)

Click the Download and Install button directly on a library card. The package is downloaded from GitHub and handed to the standard import workflow.

Install from Detail View

Open the library detail modal, review the package information, optionally select a specific version, and click the Download and Install or Update button at the bottom of the modal.

What Happens During Install

  1. The .hxlibpkg file is downloaded from the GitHub packages repository to a temporary directory
  2. If the library is an update to an existing installation, a breaking changes check is performed (see below)
  3. The downloaded package is handed to the standard import flow, which shows the pre-install preview modal
  4. The remainder of the process is identical to a normal library import: signature verification, file extraction, COM registration, integrity hashing, and database recording

Breaking Changes Detection

When installing an update to an already-installed library, Library Manager automatically compares the public HSL function signatures between the installed version and the incoming package. It detects:

  • Removed functions — public functions that existed in the old version but are absent from the new version
  • Changed signatures — functions where the parameter count, parameter types, parameter names, pass-by-reference status, or return type have changed

If breaking changes are detected, a confirmation modal is displayed listing all affected functions with their old and new signatures. You must type I ACCEPT to proceed with the installation. This safeguard ensures you are aware that existing VENUS methods using these functions may need to be updated.

Dependency Resolution

When installing a library that declares dependencies, Library Manager checks whether each dependency is already installed. If any dependencies are missing, a dependency confirmation modal is displayed showing:

  • The primary library you selected
  • A full recursive dependency tree with install status for each library:
    • Installed (green checkmark) — already installed on your system
    • Needs install (warning) — not installed but available in the store
    • Not in store (grey) — not installed and not available for automatic download

Click Download All and Install to download and install all missing dependencies in the correct order (dependencies first, then the primary library). Each package goes through the full import flow sequentially.

Note: Circular dependencies are handled automatically. The dependency resolution algorithm tracks visited libraries to prevent infinite loops.

Ratings and Reviews

The Library Store includes a community rating and review system. Star ratings are displayed on both the library cards and in the detail view.

Viewing Reviews

Open a library's detail view and scroll to the Reviews section to see community reviews with star ratings and comments.

Submitting a Review

Click the Write a Review button in the detail view to submit your own rating and comment. Reviews include your Windows username and basic system information for context.

Refreshing the Catalog

Click the Refresh button in the store toolbar to reload the catalog from GitHub. This clears the cached catalog and ratings data and fetches the latest available packages.

GitHub Repository

The Library Store is backed by a public GitHub repository at:

https://github.com/zdmilot/Library-Manager-Packages

Click the repository link in the store toolbar to open the repository in your web browser. The repository contains:

  • catalog.json — the package catalog that Library Manager reads
  • packages/<LibraryName>/ — subdirectories containing .hxlibpkg package files

A GitHub Actions workflow automatically rebuilds catalog.json whenever packages are added or updated in the repository.

Network Requirements

The Library Store requires an internet connection to:

  • Fetch the package catalog from GitHub (HTTPS)
  • Download .hxlibpkg package files from GitHub (HTTPS)
  • Load and submit community ratings and reviews

If the catalog cannot be loaded (network error, timeout, or invalid response), an error message is displayed with the option to retry.

CLI: export-archive
Library Manager » CLI Reference » export-archive

Description

Bundles one or more installed libraries into a single .hxlibarch archive file. Each library is re-packed from its current disk state, so the archive reflects any post-install modifications. System libraries and deleted libraries are automatically excluded. Each inner .hxlibpkg is individually signed with an HMAC-SHA256 integrity signature. Optionally, each inner package can be code-signed with an Ed25519 publisher certificate using --sign-key.

Usage

node cli.js export-archive (--all | --names <n1,n2,...> | --ids <id1,id2,...>) --output <path>

Options

FlagRequiredDescription
--allRequired*Export every non-deleted, non-system installed library. *Use one of --all, --names, or --ids.
--names <n1,n2,...>Required*Comma-separated library names to export
--ids <id1,id2,...>Required*Comma-separated library database IDs to export
--output <path>RequiredOutput .hxlibarch file path
--sign-key <path>NoPath to an Ed25519 private key PEM file for code signing. When provided, each inner .hxlibpkg receives both HMAC and Ed25519 digital signatures.
--sign-cert <path>NoPath to the publisher .cert.json certificate. Only needed if the certificate filename does not match the key filename. Requires --sign-key.
--db-path <dir>NoOverride the user data directory path

Archive Contents

The output archive contains:

  • An archive_manifest.json with format version, creation date, app_version, windows_version, venus_version, library count, and library names
  • One <LibraryName>.hxlibpkg entry per selected library, each containing its own manifest.json (with lineage and environment stamps), payload files, and HMAC-SHA256 signature. If --sign-key is provided, each inner package also receives an Ed25519 digital signature with the publisher certificate embedded.

Examples

:: Full backup of all installed libraries
node cli.js export-archive --all --output backup\all-libs.hxlibarch

:: Named subset
node cli.js export-archive --names "LibA,LibB" --output release\subset.hxlibarch

:: By database IDs
node cli.js export-archive --ids "abc123,def456" --output out\specific.hxlibarch

:: Code-signed archive (Ed25519)
node cli.js export-archive --all --output backup\all-libs.hxlibarch --sign-key publisher.key.pem

Handling Skipped Libraries

When using --names or --ids, libraries that are not found, deleted, or are system libraries are skipped with a warning printed to stderr. The export continues with any remaining valid libraries. If no valid libraries remain, the command fails.

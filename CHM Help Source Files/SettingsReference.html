Settings Reference
Library Manager » Settings Reference

Overview

The Settings modal provides centralized configuration for all application preferences. Open it from Overflow Menu → Settings (vertical ellipsis button in the top-right navigation bar).

Software Update

The Software Update section shows current application and system information and provides update management.

Version Information

The top of the section displays:

  • Version — Currently installed application version
  • Build — Build identifier
  • OS — Windows edition and version
  • VENUS — Detected Hamilton VENUS version

Update Check

Click Check for Updates to query the GitHub releases page for a newer version. If an update is available, the section changes to show the new version number and a View Update button that opens the update detail modal with release notes and an Update Now button.

Update Settings

SettingDefaultDescription
Automatically check for updates on startupOnWhen enabled, Library Manager checks GitHub for new releases each time the application starts. If an update is found, the update badge and View Update button appear in Settings.
Show release notes after updatesOnWhen enabled, a release notes overlay is displayed the first time the application launches after an update has been installed.

Update Process

  1. Click View Update (or Check for Updates if not yet checked)
  2. Review the release notes in the update modal
  3. Click Update Now to download the installer
  4. A full-screen progress overlay shows the download percentage
  5. Once downloaded, the installer is launched automatically and the application closes
  6. The installer runs and restarts Library Manager with the new version

Note: Updates are only available for installed versions of Library Manager. If you are running a portable or development build, the update controls are disabled and a "Not available (portable version)" message is shown.

Installation

SettingDefaultDescription
Show preview and confirmation before installing a libraryOnWhen enabled, a preview modal is displayed before each library import showing metadata, file lists, install paths, and signature verification status. When disabled, libraries are installed immediately after file selection.

Audit Trail

These settings control whether additional accountability information is required when performing destructive or impactful actions such as deleting a library, performing a rollback, overwriting an existing library, or making archive overwrite decisions.

SettingDefaultDescription
Require comment for destructive and impactful actionsOffWhen enabled, a comment field is presented before the action is executed. The comment is recorded in the audit history alongside the event details.
Require user signature for destructive and impactful actionsOffWhen enabled, the operator must type their Windows username to confirm the action. The signature is recorded in the audit history for traceability.

Actions are always logged in the event history regardless of these settings. These options add additional metadata to each audit record for environments that require formal accountability.

See Library Audit Log for details on the audit trail system.

Display

SettingDefaultDescription
Hide system libraries on the front pageOffWhen enabled, Hamilton system libraries are hidden from the All tab. System libraries with integrity warnings or errors are still shown regardless of this setting. The System tab remains accessible for viewing all system libraries.

Data Location

Displays the path to the local/ directory where all application data is stored, including settings, the library registry database, audit trail, package backups, and exports. This location is globally accessible to all users on the PC.

Click Open Folder to open the data directory in Windows Explorer.

See File Storage & Data Management for details on the directory layout.

Unsigned Libraries

SettingDefaultDescription
Include existing unsigned libraries on the front pageOffWhen enabled, scans the VENUS Library folder for .hsl and .smt files that are not tracked by any installed or system library. Discovered files appear in the Unsigned tab and can be registered, exported as packages, or removed. Files and folders starting with ~ or . are automatically excluded.

When enabled, a Scan Now button appears. Click it to perform an immediate scan. A spinning indicator is shown during the scan and a green checkmark appears on completion.

See Unsigned Libraries for details on the discovery and management workflow.

Code Signing

Configure a default publisher identity for signing exported packages. Signed packages allow recipients to verify the publisher's identity and detect tampering.

Key and Certificate Selection

FieldDescription
Private Key (.key.pem)Browse to select an Ed25519 private key file. This key is used to sign packages during export.
Certificate (.cert.json)Browse to select the corresponding publisher certificate file. The certificate contains the public key, holder name, and organization.

When both files are configured and valid, a publisher summary is displayed showing the holder name, key ID, and organization.

Generate Key Pair

Click Generate Key Pair... to create a new Ed25519 signing key pair. You will be asked to select an output directory and provide holder information. Two files are generated:

  • <name>.key.pem — Private key (keep secure, do not share)
  • <name>.cert.json — Public certificate (distribute with packages)

Clear

Click Clear to remove the configured signing key and certificate. This does not delete the files from disk.

See Code Signing for the full technical documentation.

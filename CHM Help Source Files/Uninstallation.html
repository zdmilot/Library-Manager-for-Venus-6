Uninstallation
Library Manager » Uninstallation

Overview

The Library Manager uninstaller provides a three-tier removal system that lets you choose exactly how much data to remove along with the application. When you launch the uninstaller, a dialog appears with three options before any files are deleted.

Launching the Uninstaller

You can start the uninstall process in any of the following ways:

  • Start Menu — open the Library Manager program group and click Uninstall Library Manager
  • Windows Settings — go to Settings → Apps → Installed apps, locate Library Manager, and click Uninstall
  • Control Panel — open Programs and Features, select Library Manager, and click Uninstall

Removal Mode Selection

Before anything is removed, a dialog asks you to choose one of three removal modes:

1. Application Only (least destructive)

Removes only the core application files:

  • Executable (Library Manager.exe)
  • Runtime DLLs (nw.dll, node.dll, etc.)
  • Dependencies (node_modules\)
  • UI assets (html\, assets\, icons\, locales\)
  • CLI files, help file, README, and legal documents

Keeps all user data, including:

  • Application settings and configuration (local\settings.json)
  • Library registry, groups, and navigation tree
  • Package archives (local\packages\)
  • Exported files (local\exports\)
  • Audit trail (local\audit_trail.json)
  • Publisher registry
  • Database templates and verification hashes (db\)

When to use: You plan to reinstall the application later and want to pick up exactly where you left off — all settings, packages, and history will be restored on reinstall.

2. Standard

Removes the core application files (same as Application Only) plus application settings and configuration files:

RemovedKept
  • local\settings.json
  • local\installed_libs.json
  • local\groups.json
  • local\tree.json
  • local\links.json
  • local\unsigned_libs.json
  • local\publisher_registry.json
  • db\settings.json
  • db\installed_libs.json
  • db\groups.json
  • db\tree.json
  • db\links.json
  • db\unsigned_libs.json
  • Package archives (local\packages\)
  • Exported files (local\exports\)
  • Audit trail (local\audit_trail.json)
  • System library definitions (db\system_libraries.json)
  • File verification hashes (db\system_library_hashes.json)

When to use: You are replacing the application with a different program, but want to keep important records — package archives, audit logs, event history, and the verification data (hashes) needed to validate retained archives and logs.

3. Full Removal (most destructive)

WARNING: Destructive action. This option removes the application and all associated data, including:

  • All audit files, event history, and audit trail logs
  • All archived library packages (local\packages\)
  • All exported files (local\exports\)
  • All backups and retained verification data
  • All settings, configuration, and database files

When you select this option, an additional confirmation dialog appears asking you to verify that you understand the consequences. You must click Yes to proceed.

Important: Full Removal does not delete anything in external VENUS directories. Any libraries already installed in the Hamilton Library\ or Methods\ directories will remain intact and continue to function in VENUS.

When to use: You want a completely clean slate with no trace of the application or its data on disk.

What Happens During Uninstall

  1. The Removal Mode Selection dialog is displayed. Choose a tier and click Uninstall, or click Cancel to abort.
  2. If Full Removal is selected, an additional confirmation dialog warns about the destructive consequences and asks for explicit approval.
  3. The built-in Inno Setup uninstaller removes tracked application files (binaries, DLLs, UI assets, runtime dependencies). Data files (in local\ and db\) are not removed at this stage because they are flagged for manual control.
  4. After the core uninstall completes, a post-uninstall cleanup step runs according to the selected removal mode:
    • Application Only — no additional cleanup; all data directories remain.
    • Standard — settings and configuration JSON files are deleted from both local\ and db\. The db\ directory is removed only if it becomes empty (verification hashes and system library definitions may still reside there).
    • Full Removal — the entire local\ and db\ directory trees are deleted, and the install directory itself is removed if empty.
  5. Start Menu entries and the desktop shortcut (if created) are removed regardless of which tier is selected.

Residual Files After Uninstall

Depending on the removal mode, some files and directories may remain on disk after uninstalling:

ModeResidual Contents
Application Only The entire local\ and db\ directories remain intact. The install directory is preserved.
Standard local\packages\, local\exports\, local\audit_trail.json, and (within db\) system_libraries.json and system_library_hashes.json. The install directory is preserved.
Full Removal None. The install directory is removed entirely (assuming no external files were manually placed there).

Reinstalling After Uninstall

If the Application Only or Standard mode was used, reinstalling Library Manager to the same directory will automatically detect and reuse the retained data files. The application will launch with your preserved packages, audit history, and (in Application Only mode) your settings intact.

Silent / Unattended Uninstall

The uninstaller supports Inno Setup’s standard silent switches. In silent mode, the removal mode selection dialog is not shown and the uninstaller defaults to Application Only (least destructive) behavior:

unins000.exe /SILENT
unins000.exe /VERYSILENT /SUPPRESSMSGBOXES

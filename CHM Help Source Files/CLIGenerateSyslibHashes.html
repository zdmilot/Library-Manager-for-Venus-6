CLI: generate-syslib-hashes
Library Manager » CLI Reference » generate-syslib-hashes

Description

Generates an integrity baseline file for Hamilton system libraries. The baseline captures the Hamilton metadata footer values ($$valid$$, $$checksum$$, $$author$$, etc.) from each HSL-type source file on a known-good installation. Binary files and non-HSL assets are skipped.

The resulting baseline JSON file is used by verify-syslib-hashes to detect tampering or corruption.

Usage

node cli.js generate-syslib-hashes --source-dir <path> [--output <path>]

Options

FlagRequiredDescription
--source-dir <path>RequiredPath to a known-good Library folder (e.g., a fresh VENUS installation)
--output <path>OptionalOutput file path. Defaults to db/system_library_hashes.json

Strategy: Hamilton Metadata Footer

Unlike user libraries which use SHA-256 file hashing, system libraries use Hamilton's proprietary metadata footer. Only files with extensions .hsl, .hs_, and .smt are tracked, and only when they contain a valid metadata footer. See HSL File Architecture for full footer format details.

For each tracked file, the baseline stores:

FieldDescription
validThe $$valid$$ flag value (1 = unmodified, 0 = edited)
checksumThe $$checksum$$ value
authorThe $$author$$ value
timeThe $$time$$ value
lengthThe $$length$$ value

System Libraries Source

The command reads the list of system libraries and their known files from db/system_libraries.json. This JSON array defines each system library with its _id, canonical_name, and discovered_files list.

Process Steps

  1. Loads system_libraries.json to get the list of system libraries and their files
  2. Iterates each library's discovered files
  3. Skips binary files (non-.hsl, .hs_, .smt extensions)
  4. Parses the metadata footer of each HSL-type file
  5. Files without a footer are noted but not tracked
  6. Missing files are warned about but do not stop the process
  7. Writes the baseline JSON to the output path

Baseline File Format

{
  "_meta": {
    "generated_at": "2024-06-01T14:30:00.000Z",
    "source_dir": "C:\\Program Files (x86)\\HAMILTON\\Library",
    "strategy": "hamilton-footer",
    "hsl_extensions": [".hsl", ".hs_", ".smt"],
    "description": "Integrity baseline for Hamilton system libraries..."
  },
  "libraries": {
    "HSLAppsLib": {
      "_id": "...",
      "files": {
        "HSLAppsLib.hsl": {
          "valid": 1,
          "checksum": "abc123...",
          "author": "HAMILTON",
          "time": "2024-01-15 10:30:00",
          "length": "48210"
        }
      }
    }
  }
}

Output Summary

After generation, the command prints a summary:

Generating system library baseline from: C:\Program Files (x86)\HAMILTON\Library
System libraries defined: 85
Strategy: Hamilton metadata footer ($$valid$$/$$checksum$$) only.

Baselined 142 / 310 files across 72 libraries.
  Footer tracked  : 142 files
  Binary skipped  : 156 files

Note: 12 HSL-type file(s) had no metadata footer (not tracked):
  [SomeLib] SomeFile.hsl

Baseline file written: db/system_library_hashes.json

Examples

:: Generate from default HAMILTON Library path
node cli.js generate-syslib-hashes --source-dir "C:\Program Files (x86)\HAMILTON\Library"

:: Generate from a backup and save to a custom location
node cli.js generate-syslib-hashes --source-dir "D:\Venus-Backup\Library" --output baseline.json

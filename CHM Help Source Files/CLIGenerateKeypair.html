CLI: generate-keypair
Library Manager » CLI Reference » generate-keypair

Description

Generates an Ed25519 key pair for code signing library packages. The resulting private key (.key.pem) is used to sign packages during creation (with create-package --sign-key), while the publisher certificate (.cert.json) is distributed to package consumers.

Usage

node cli.js generate-keypair --publisher <name> [options]

Options

FlagRequiredDescription
--publisher <name>RequiredPublisher name (3–500 characters)
--organization <name>OptionalOrganization name
--output-dir <dir>OptionalOutput directory for key files. Default: current working directory
--forceOptionalOverwrite existing key/cert files
--author-password <pw>OptionalRequired when the publisher name is a restricted OEM name

Output Files

Two files are generated in the output directory:

  • <publisher>.key.pem — Ed25519 private key (PEM-encoded PKCS#8). Keep this secret.
  • <publisher>.cert.json — Publisher certificate (JSON). Distribute freely to package consumers.

Certificate Format

{
  "cert_format":   "hxlibpkg-publisher-cert/1.0",
  "publisher":     "Jane Smith",
  "organization":  "Acme Pharma",
  "public_key":    "<base64 raw Ed25519 public key>",
  "fingerprint":   "<SHA-256 hex of raw public key bytes>",
  "key_id":        "<first 16 chars of fingerprint>",
  "created_date":  "<ISO 8601 timestamp>"
}

Examples

:: Generate a key pair
node cli.js generate-keypair --publisher "Jane Smith" --organization "Acme Pharma"

:: Generate to specific directory
node cli.js generate-keypair --publisher "Jane Smith" --output-dir C:\Keys

:: Overwrite existing keys
node cli.js generate-keypair --publisher "Jane Smith" --force

Code Signing Workflow

  1. Generate a key pair with generate-keypair
  2. Create signed packages with create-package --sign-key <key.pem>
  3. Distribute the .cert.json to package consumers
  4. On import, packages signed with a known OEM publisher name display an OEM Verified badge

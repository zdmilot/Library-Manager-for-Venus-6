Package Formats
Venus Library Manager » Package Formats

Overview

Venus Library Manager uses two ZIP-based container formats for distributing and archiving Hamilton VENUS libraries. Both formats use standard ZIP compression and can be inspected with any ZIP tool, but they contain a structured internal layout that Venus Library Manager requires.

Single Library Package (.hxlibpkg)

A .hxlibpkg file encapsulates one complete library: its source files, demo methods, icon, and all metadata needed to install and manage the library.

Internal Structure

MyLibrary.hxlibpkg (ZIP archive)
  ├── manifest.json            <-- Required: library metadata
  ├── signature.json           <-- Automatic: HMAC-SHA256 integrity signature
  ├── library/                 <-- Library payload files
  │   ├── MyLibrary.hsl
  │   ├── MyLibrary.hs_
  │   ├── MyLibrary.stp
  │   ├── MyLibrary.bmp
  │   └── MyLibrary.chm
  ├── demo_methods/            <-- Optional: demo/example methods
  │   ├── Demo_MyLibrary.hsl
  │   └── Demo_MyLibrary.lay
  └── icon/                    <-- Optional: library icon image
      └── MyLibrary.png

Manifest (manifest.json)

Every .hxlibpkg must contain a manifest.json at the archive root. This file holds all metadata and the file inventory. Missing or invalid manifests cause the import to fail with an error.

FieldTypeRequiredDescription
format_versionstringNoPackage format version (currently "1.0")
library_namestringYesName of the library (used as the install directory name)
authorstringYesLibrary author's name
organizationstringNoAuthor's organization or company
versionstringYesLibrary version string (e.g., "1.0.0")
venus_compatibilitystringNoCompatible VENUS version range (e.g., "4.7+")
descriptionstringNoHuman-readable description of the library
tagsstring[]NoSearchable keyword tags
created_datestringNoISO 8601 timestamp of package creation
library_imagestringNoFilename of the library icon image
library_image_base64stringNoBase64-encoded raw library image data (no overlay). The composited icon with the Library Manager logo overlay is stored separately in the icon/ directory for Windows file-system display only (see Visual Identity)
library_image_mimestringNoMIME type of the library image
library_filesstring[]YesBasenames of all library payload files (including CHMs)
demo_method_filesstring[]NoBasenames of demo method files
help_filesstring[]NoBasenames of .chm help files
com_register_dllsstring[]NoBasenames of DLLs requiring COM registration

Example Manifest

{
  "format_version": "1.0",
  "library_name": "MyVenusLibrary",
  "author": "Jane Smith",
  "organization": "Acme Pharma",
  "version": "1.0.0",
  "venus_compatibility": "4.7+",
  "description": "High-level pipetting helpers for the ML600.",
  "tags": ["pipetting", "ML600"],
  "created_date": "2026-01-15T10:30:00.000Z",
  "library_image": "MyVenusLibrary.png",
  "library_image_base64": "iVBORw0KGgo...",
  "library_image_mime": "image/png",
  "library_files": [
    "MyVenusLibrary.hsl",
    "MyVenusLibrary.hs_",
    "MyVenusLibrary.stp",
    "MyVenusLibrary.bmp"
  ],
  "demo_method_files": ["Demo_MyVenusLibrary.hsl"],
  "help_files": ["MyVenusLibrary.chm"],
  "com_register_dlls": []
}

Package Signature (signature.json)

Every .hxlibpkg created by Venus Library Manager includes a signature.json entry containing an HMAC-SHA256 cryptographic signature. This signature covers every other entry in the ZIP archive and is used to verify that the package has not been corrupted or tampered with.

FieldTypeDescription
algorithmstringAlways "HMAC-SHA256"
createdstringISO 8601 timestamp of when the signature was generated
filesobjectSorted map of { "entryName": "sha256hex" } for every ZIP entry except signature.json
hmacstringHMAC-SHA256 computed over the canonical JSON representation of the files map

Example Signature

{
  "algorithm": "HMAC-SHA256",
  "created": "2026-01-15T10:30:00.000Z",
  "files": {
    "demo_methods/Demo_MyVenusLibrary.hsl": "b2c3d4e5...",
    "icon/MyVenusLibrary.png": "c3d4e5f6...",
    "library/MyVenusLibrary.hsl": "d4e5f6a7...",
    "library/MyVenusLibrary.hs_": "e5f6a7b8...",
    "manifest.json": "a1b2c3d4..."
  },
  "hmac": "9f8e7d6c..."
}

Note: Packages created before signing was added (legacy packages) do not contain a signature.json. These are treated as unsigned on import and are accepted without error — there is no forced rejection of unsigned packages.

See Integrity Verification for details on how signing relates to file-level integrity checking, and CLI: verify-package for standalone package verification.

Multi-Library Archive (.hxlibarch)

A .hxlibarch file is a ZIP archive that contains multiple .hxlibpkg files plus an archive-level manifest. This format is used for bundling, backup, and distribution of entire library sets.

Internal Structure

bundle.hxlibarch (ZIP archive)
  ├── archive_manifest.json    <-- Archive-level metadata
  ├── LibraryA.hxlibpkg        <-- Complete .hxlibpkg for LibraryA
  ├── LibraryB.hxlibpkg        <-- Complete .hxlibpkg for LibraryB
  └── LibraryC.hxlibpkg        <-- Complete .hxlibpkg for LibraryC

Archive Manifest (archive_manifest.json)

FieldTypeDescription
format_versionstringArchive format version ("1.0")
archive_typestringAlways "hxlibarch"
created_datestringISO 8601 timestamp of archive creation
library_countintegerNumber of packages in the archive
librariesstring[]List of library names included
archive_icon_base64stringBase64-encoded purple archive icon PNG
archive_icon_mimestringMIME type of the archive icon ("image/png")

In addition to the manifest fields, the archive ZIP also contains a physical icon file at icon/archive_icon.png. This purple archive icon (three stacked purple diamonds) visually identifies the file as a multi-library archive. See Visual Identity & Assets for details on how the icon is generated and stored.

File Type Reference

Library payload files can include any of the following Hamilton VENUS file types:

ExtensionDescriptionIntegrity Tracked
.hslHSL library definition (primary entry point)Yes (SHA-256 for user libs; footer for system libs)
.hs_HSL include / string table / implementationYes
.hsiHSL include (alternate convention)No
.smtStep module templateFooter tracked for system libs only
.stpStep template parametersNo
.subSub-method fileYes (SHA-256)
.dllDynamic-link library (may require COM registration)Yes (SHA-256, if COM-registered)
.bmpLibrary icon bitmapNo
.chmCompiled HTML Help fileNo

Note: During import, .chm files are automatically detected from the library files list and tracked separately in the help_files field so the GUI can provide quick-access help links.

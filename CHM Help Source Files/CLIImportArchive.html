CLI: import-archive
Venus Library Manager » CLI Reference » import-archive

Description

Extracts every .hxlibpkg embedded inside a .hxlibarch ZIP archive and installs each one sequentially. This is the batch import command for multi-library distribution bundles.

Usage

node cli.js import-archive --file <path> [options]

Options

FlagRequiredDescription
--file <path>RequiredPath to the .hxlibarch file
--lib-dir <path>NoOverride library install root
--met-dir <path>NoOverride methods (demo) install root
--forceNoOverwrite already-installed libraries without error
--no-groupNoSkip auto-assigning libraries to groups
--no-cacheNoSkip caching packages in the package store
--db-path <dir>NoOverride the user data directory path
--store-dir <dir>NoOverride the package store directory

Processing

  1. The .hxlibarch file is opened as a ZIP archive
  2. All entries ending in .hxlibpkg are identified (non-package entries are ignored)
  3. If no .hxlibpkg entries are found, the command fails
  4. Each .hxlibpkg entry is processed sequentially:
    • The inner ZIP is opened in memory and its manifest.json is read
    • If a library with the same name exists without --force, that entry is skipped as an error
    • Files are extracted, hashes computed, functions parsed, and the database record created
    • The inner package is cached in the package store
  5. An aggregate summary is printed showing success and failure counts

Examples

:: Import all libraries in a bundle
node cli.js import-archive --file bundle.hxlibarch

:: Force overwrite all
node cli.js import-archive --file bundle.hxlibarch --force

:: Use custom paths and test database
node cli.js import-archive --file bundle.hxlibarch ^
    --db-path tests\scratch-db ^
    --lib-dir tests\scratch-lib ^
    --met-dir tests\scratch-met

Exit Behavior

If any individual package fails to import, the remaining packages still attempt to install. The command exits with code 1 if one or more packages failed. Individual failure messages are printed to stderr.

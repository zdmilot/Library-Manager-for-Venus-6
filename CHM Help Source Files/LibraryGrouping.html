Library Grouping
Library Manager » Library Grouping

Overview

Library Manager supports organizing installed libraries into custom groups for easier navigation and management. Groups appear as tabs in the main window, allowing you to filter the library view to show only libraries assigned to a specific group.

Default Groups (Hardcoded)

The application defines seven built-in groups. These groups are always present and cannot be removed. They are resolved at runtime and are never stored in groups.json.

NamePurpose
AllShows all installed (non-deleted) libraries
RecentRecently imported libraries
StarredUser-favorited libraries (protected, non-deletable)
OEMProtected OEM-authored libraries
ImportImport workflow tab
ExportExport workflow tab
HistoryAction history

User-created groups are stored in groups.json with "default": false and auto-generated _id values.

OEM Default Group

The OEM group is a special protected group that ships with the application. It contains libraries authored by recognized OEM vendors (such as Hamilton and other equipment manufacturers) and is distinguished from regular custom groups in several important ways:

Protection

  • Non-deletable and non-renamable - The OEM group cannot be deleted or renamed through the settings panel.
  • Always visible - It appears in the left-side navigation between the System libraries group and user-created groups. It cannot be hidden.
  • Solid checkmark icon - Uses a solid checkmark icon to visually distinguish it from user-created groups.
  • OEM Verified badge - Individual OEM library cards display a blue verified checkmark next to the author name. Hovering the badge shows certificate details (publisher, key ID, fingerprint, algorithm, and issued date). Verified OEM libraries also include a Code Signing Certificate section in their detail modal. See Integrity Verification for full documentation.

Reserved Author Names

Certain author and organization names associated with OEM vendors are reserved and cannot be used when creating packages. This prevents unauthorized attribution of packages to established vendors. If a reserved name is entered, the application will display an error.

Empty State

When no OEM-authored packages are installed, the OEM tab displays a dedicated placeholder message: "No OEM packages installed yet. Import an OEM-authored library package to see it here."

Managing Groups

Creating a Group

Create new groups from the Settings panel. Each group has a name and an icon class. New groups are added to the navigation tree and immediately become available as filter tabs.

Renaming and Deleting Groups

Groups can be renamed or deleted from the Settings panel. Deleting a group does not delete the libraries assigned to it - they become "unassigned" and remain visible in the All view.

Reordering Groups

Drag-and-drop reordering is supported in the Settings panel. The order you set determines the tab display order in the main window.

Assigning Libraries to Groups

Automatic Assignment

When a library is imported, it is automatically assigned to the first non-default custom group. If no custom groups exist, a default group named "Libraries" is created and the library is assigned to it.

This behavior is always enabled and cannot be disabled in the GUI. CLI imports can bypass it with the --no-group flag.

Manual Assignment

In the Settings panel, libraries can be drag-and-dropped between groups. An "Unassigned Libraries" pseudo-group shows all libraries that are not currently in any group.

Group Visibility

Groups can be shown or hidden in the navigation using the favorite/show-hide toggle. Hidden groups and their assigned libraries still exist but are not displayed as navigation tabs.

Data Persistence

Default groups (All, Recent, Starred, OEM, Import, Export, History) are not stored in groups.json. They are defined within the application and resolved in-memory. Only user-created custom groups are stored in groups.json.

Group-to-library assignments are stored in tree.json within the user data directory. See Data Model & Persistence for details.

Unsigned Group

When the Unsigned Libraries feature is enabled and at least one unsigned library has been discovered by scanning, a special Unsigned group is injected into the navigation bar. This group is distinct from both default and custom groups:

  • Uses the far fa-times-circle icon
  • Appears after the OEM group in the left navigation
  • Locked - libraries cannot be dragged into or out of it
  • Not stored in tree.json or groups.json - it is generated dynamically
  • Not included in the sortable group ordering
  • Hidden automatically when the feature is disabled or no unsigned libraries exist

See Unsigned Libraries for full details on enabling and using this feature.

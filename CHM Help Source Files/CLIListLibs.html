CLI: list-libs
Library Manager » CLI Reference » list-libs

Description

Prints a summary of every installed library record in the database. For each library, displays the database ID, name, version, author, tags, install paths, install date, COM DLL list, and public function signatures parsed from .hsl files.

Usage

node cli.js list-libs [options]

Options

FlagRequiredDescription
--include-deletedNoInclude soft-deleted library entries in the output. By default, deleted libraries are hidden.
--json, -jNoOutput the full library records as a raw JSON array. Useful for scripting and piping to other tools.
--db-path <dir>NoOverride the user data directory path

Output Format

In human-readable mode, each library is printed with labeled fields followed by a separator line. Public function signatures are listed with their qualified names, parameter types, and return types.

In JSON mode (--json), the full database record array is printed, including all metadata, file lists, hashes, and parsed function data.

Examples

:: Human-readable summary
node cli.js list-libs

:: JSON output for scripting
node cli.js list-libs --json

:: Include deleted libraries
node cli.js list-libs --include-deleted --json

:: Against a specific database
node cli.js list-libs --db-path D:\test\scratch-db

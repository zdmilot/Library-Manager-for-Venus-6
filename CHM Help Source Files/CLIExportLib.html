CLI: export-lib
Library Manager » CLI Reference » export-lib

Description

Rebuilds a .hxlibpkg from the currently installed files on disk for a single named library. This is a mutable export - it captures the current on-disk state, including any changes made since the original import. The exported package is automatically signed with an HMAC-SHA256 integrity signature. Optionally, the package can be code-signed with an Ed25519 publisher certificate using --sign-key.

Usage

node cli.js export-lib (--name <name> | --id <id>) --output <path>

Options

FlagRequiredDescription
--name <name>Required*Library name (as shown in list-libs). *Use either --name or --id.
--id <id>Required*Library database ID. *Use either --name or --id.
--output <path>RequiredOutput .hxlibpkg file path
--sign-key <path>NoPath to an Ed25519 private key PEM file for code signing. When provided, the exported package receives both HMAC and Ed25519 digital signatures. The matching .cert.json is auto-detected from the key filename unless --sign-cert is specified.
--sign-cert <path>NoPath to the publisher .cert.json certificate. Only needed if the certificate filename does not match the key filename. Requires --sign-key.
--db-path <dir>NoOverride the user data directory path

Export Process

  1. The library record is looked up in the database by name or ID
  2. Validation checks:
    • System libraries cannot be exported (SYSTEM_LIBRARY_READ_ONLY error)
    • Deleted libraries cannot be exported
    • All expected library files must exist on disk (aborts if any are missing)
  3. A new manifest.json is built from the current database record with a fresh created_date, the current app_version, windows_version, and venus_version
  4. An "exported" event is appended to the existing package_lineage from the database record, recording the operator, hostname, and environment of the exporting machine
  5. Any forward-compatible fields in the database record that are not part of the known schema are preserved in the exported manifest
  6. All library files and help files are packed into library/
  7. All demo method files are packed into demo_methods/
  8. If --sign-key is provided, the ZIP is code-signed with an Ed25519 digital signature and the publisher certificate is embedded in signature.json. Otherwise, the ZIP is signed with HMAC-SHA256 only.
  9. The ZIP is wrapped in a binary container and written to the specified output path

Examples

:: Export by name
node cli.js export-lib --name "MyLibrary" --output MyLibrary.hxlibpkg

:: Export by database ID
node cli.js export-lib --id a1b2c3d4 --output dist\MyLibrary.hxlibpkg

:: Export with Ed25519 code signing
node cli.js export-lib --name "MyLibrary" --output MyLibrary.hxlibpkg --sign-key publisher.key.pem

:: Export with explicit certificate path
node cli.js export-lib --name "MyLibrary" --output MyLibrary.hxlibpkg --sign-key keys/publisher.key.pem --sign-cert certs/publisher.cert.json

Error Conditions

  • Neither --name nor --id specified -> error
  • --output not specified -> error
  • Library not found in database -> error
  • Library is a system library -> SYSTEM_LIBRARY_READ_ONLY error
  • Library is deleted -> error
  • Library file missing on disk -> error with path shown

Visual Identity & Assets
Library Manager for Venus 6 » Visual Identity & Assets

Overview

Library Manager for Venus 6 uses a consistent visual branding system across packages and archives. All visual assets are pre-rendered and stored in the assets/ directory to avoid runtime conversion overhead and ensure reliable rendering across environments.

Assets Folder

The assets/ directory contains the following pre-rendered PNG files:

FileDimensionsPurpose
app_logo_gray_128.png128 × 128 pxGrayscale version of the application logo, used as a brand overlay during package icon compositing
archive_icon_64.png64 × 64 pxPurple archive icon (small resolution)
archive_icon_128.png128 × 128 pxPurple archive icon, embedded into .hxlibarch exports as the archive-level icon
archive_icon_256.png256 × 256 pxPurple archive icon (large resolution)

Asset Generation

The static assets are generated from source files using two methods:

Archive Icon (archive_icon_*.png)

Rendered from ArchiveIcon.svg using the Inkscape command-line interface at 64, 128, and 256 pixel resolutions. The SVG depicts three stacked purple diamonds representing a multi-library bundle. Example generation command:

inkscape ArchiveIcon.svg --export-type=png --export-width=128 --export-filename=assets/archive_icon_128.png

Grayscale Logo (app_logo_gray_128.png)

Generated from the application's full-colour logo using Python's Pillow library. The process applies full desaturation (grayscale conversion) and resizes the image to 128 × 128 pixels. Example generation script:

from PIL import Image
img = Image.open("icons/app_logo.png").convert("L").convert("RGBA").resize((128, 128))
img.save("assets/app_logo_gray_128.png")

These pre-rendered PNGs replace earlier runtime approaches (SVG-to-Canvas rendering and live grayscale conversion), reducing code complexity and improving reliability.

Package Icon Compositing

When a .hxlibpkg is built through the GUI Library Packager, a composited icon is generated and written only to the icon/ directory inside the package ZIP. This composited icon is what Windows displays for the .hxlibpkg file in Explorer. The package manifest's library_image_base64 field stores the raw, uncomposited library image so that imported libraries display the original artwork without any overlay.

With a User-Supplied Image

  1. The user's chosen library image is drawn at full size on an HTML5 Canvas
  2. The grayscale Library Manager logo (app_logo_gray_128.png) is loaded from the assets folder
  3. The grayscale logo is overlaid in the bottom-right corner of the canvas at one-third of the canvas dimensions
  4. The resulting composited image is exported as a base64-encoded PNG and written to the icon/ directory in the ZIP

The compositing process produces a branded file-system icon that prominently shows the library's own artwork with a small grayscale Library Manager logo overlaid in the bottom-right corner at one-third of the canvas dimensions. This is similar to how Windows draws shortcut arrows on desktop shortcuts — the overlay clearly identifies the file as a Library Manager for Venus 6 package without obscuring the library's visual identity. The composited icon is used only for the file-system representation; the package manifest stores the raw, uncomposited library image for display inside the application.

Without a User-Supplied Image

If no image was selected by the user and no matching .bmp was auto-detected from the library files, the grayscale Library Manager logo is drawn at full canvas size and used as the file-system icon. The manifest will contain no image data in this case.

Implementation

The compositing is performed by the pkgCompositeLibraryIcon() function in main.js. It uses the HTML5 Canvas API (<canvas> element) to draw and composite the images, and returns the result as a base64-encoded PNG. The composited result is used exclusively for the icon/ ZIP entry; the manifest retains the original library image.

Archive Icon

Multi-library archive files (.hxlibarch) embed a distinctive purple archive icon that visually identifies the file as containing multiple library packages. The icon depicts three stacked purple diamonds.

How It Is Embedded

When an archive is exported:

  1. The static asset assets/archive_icon_128.png is read from disk using synchronous file I/O
  2. The PNG data is base64-encoded and stored in the archive_manifest.json file under the archive_icon_base64 and archive_icon_mime fields
  3. The PNG file is also written as a physical file to icon/archive_icon.png inside the archive ZIP

Implementation

The archive icon is loaded by the getArchiveIconPng() function in main.js, which returns a Node.js Buffer containing the raw PNG data. This function reads the file synchronously from the assets directory.

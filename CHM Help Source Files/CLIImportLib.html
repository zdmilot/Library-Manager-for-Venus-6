CLI: import-lib
Library Manager » CLI Reference » import-lib

Description

Reads a .hxlibpkg file, verifies its package signature, validates its manifest.json, extracts payload files to the configured VENUS library and demo-methods directories, computes integrity hashes, parses public HSL functions, registers the library record in the database, and caches the package in the package store for future rollback.

Usage

node cli.js import-lib --file <path> [options]

Options

FlagRequiredDescription
--file <path>RequiredPath to the .hxlibpkg file to import
--lib-dir <path>NoOverride library install root directory. Default: from DB settings, fallback C:\Program Files (x86)\HAMILTON\Library
--met-dir <path>NoOverride methods (demo) install root directory. Default: from DB settings, fallback ...\Methods
--forceNoOverwrite an already-installed library without returning an error. Also overrides package signature verification failure.
--no-groupNoSkip auto-assigning the new library to a group
--no-cacheNoSkip caching the package in the package store
--db-path <dir>NoOverride the user data directory path
--store-dir <dir>NoOverride the package store directory
--author-password <pw>NoAuthorize importing packages with a restricted OEM or laboratory automation company author name

Import Process

  1. The .hxlibpkg file is opened as a ZIP archive
  2. Signature verification: If a signature.json is present, the package's integrity signature is verified. For v2.0 packages, this includes Ed25519 digital signature verification and publisher certificate validation. If the package is code-signed by a recognized OEM publisher, an OEM Verified badge is shown. If verification fails, the command exits with an error unless --force is set. Unsigned (legacy) packages are accepted with a notice.
  3. manifest.json is read and validated (missing manifest = error)
  4. Version mismatch check: If the package's app_version differs from the running Library Manager version, a warning is printed. A stronger warning is issued when the package was created by a newer major version. Legacy packages (no app_version) are noted in the console.
  5. If a library with the same name exists and is not deleted, the command fails unless --force is set
  6. Files in library/ are extracted to <lib-dir>\<LibraryName>
  7. Files in demo_methods/ are extracted to <met-dir>\Library Demo Methods\<LibraryName>
  8. Files in help_files/ (legacy) are extracted to the library directory
  9. .chm files are auto-detected and tracked in the help_files field
  10. SHA-256 integrity hashes are computed for tracked file types
  11. Public functions are parsed from .hsl files
  12. The database record is created (or replaced if overwriting), including app_version, format_version, windows_version, venus_version, and package_lineage from the manifest. Any unrecognized manifest fields are preserved for forward compatibility.
  13. The library is auto-assigned to a group (unless --no-group)
  14. The package file is cached in the package store (unless --no-cache)

Examples

:: Basic import
node cli.js import-lib --file MyLibrary.hxlibpkg

:: Force overwrite existing
node cli.js import-lib --file MyLibrary.hxlibpkg --force

:: Custom install paths
node cli.js import-lib --file MyLibrary.hxlibpkg ^
    --lib-dir D:\Hamilton\Library ^
    --met-dir D:\Hamilton\Methods

:: Import without caching
node cli.js import-lib --file MyLibrary.hxlibpkg --no-cache

Output

On success, prints the library name, file count, install paths, and cache location. If the library requires COM DLLs, a reminder to register them manually is printed.

Error Conditions

  • --file not specified -> error
  • File does not exist -> error
  • Invalid package (no manifest.json) -> error
  • Library already installed without --force -> error
  • Signature verification failed without --force -> error
  • Invalid library name (path separators, reserved characters, .., trailing dots/spaces) -> error
  • Author name shorter than 3 or longer than 29 characters -> error
  • Organization name shorter than 3 or longer than 29 characters (if provided) -> error
  • Restricted OEM author/organization without --author-password -> error
  • Incorrect --author-password for restricted author -> error

CLI: import-lib
Venus Library Manager » CLI Reference » import-lib

Description

Reads a .hxlibpkg file, validates its manifest.json, extracts payload files to the configured VENUS library and demo-methods directories, computes integrity hashes, parses public HSL functions, registers the library record in the database, and caches the package in the package store for future rollback.

Usage

node cli.js import-lib --file <path> [options]

Options

FlagRequiredDescription
--file <path>RequiredPath to the .hxlibpkg file to import
--lib-dir <path>NoOverride library install root directory. Default: from DB settings, fallback C:\Program Files (x86)\HAMILTON\Library
--met-dir <path>NoOverride methods (demo) install root directory. Default: from DB settings, fallback ...\Methods
--forceNoOverwrite an already-installed library without returning an error
--no-groupNoSkip auto-assigning the new library to a group
--no-cacheNoSkip caching the package in the package store
--db-path <dir>NoOverride the user data directory path
--store-dir <dir>NoOverride the package store directory

Import Process

  1. The .hxlibpkg file is opened as a ZIP archive
  2. manifest.json is read and validated (missing manifest = error)
  3. If a library with the same name exists and is not deleted, the command fails unless --force is set
  4. Files in library/ are extracted to <lib-dir>\<LibraryName>
  5. Files in demo_methods/ are extracted to <met-dir>\Library Demo Methods\<LibraryName>
  6. Files in help_files/ (legacy) are extracted to the library directory
  7. .chm files are auto-detected and tracked in the help_files field
  8. SHA-256 integrity hashes are computed for tracked file types
  9. Public functions are parsed from .hsl files
  10. The database record is created (or replaced if overwriting)
  11. The library is auto-assigned to a group (unless --no-group)
  12. The package file is cached in the package store (unless --no-cache)

Examples

:: Basic import
node cli.js import-lib --file MyLibrary.hxlibpkg

:: Force overwrite existing
node cli.js import-lib --file MyLibrary.hxlibpkg --force

:: Custom install paths
node cli.js import-lib --file MyLibrary.hxlibpkg ^
    --lib-dir D:\Hamilton\Library ^
    --met-dir D:\Hamilton\Methods

:: Import without caching
node cli.js import-lib --file MyLibrary.hxlibpkg --no-cache

Output

On success, prints the library name, file count, install paths, and cache location. If the library requires COM DLLs, a reminder to register them manually is printed.

Error Conditions

  • --file not specified → error
  • File does not exist → error
  • Invalid package (no manifest.json) → error
  • Library already installed without --force → error

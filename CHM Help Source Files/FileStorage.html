File Storage & Data Management
Library Manager » File Storage & Data Management

Overview

Library Manager stores all persistent application data in a single local/ directory within the application’s own install folder under Program Files (x86). This design ensures that all settings, library registries, audit trails, package backups, and exports are globally accessible to every user account on the PC — which is essential because VENUS libraries must span the entire workstation and are not per-user resources.

Installation Directory

The application is installed to:

C:\Program Files (x86)\Library Manager\

This uses the 32-bit Program Files directory (x86) because the application is built on a 32-bit architecture (NW.js ia32). The install directory is used for application binaries and static assets.

Local Data Directory

All mutable application data is stored under:

C:\Program Files (x86)\Library Manager\local\

Directory Structure

local/groups.jsonJSON fileUser-created custom library groups (name, icon, color).

local/links.jsonJSON fileVENUS tool shortcuts and folder path references (Method Editor, Labware Editor, Library folder path, etc.).
local/unsigned_libs.jsonJSON fileRegistry of unsigned HSL files discovered during scans of the VENUS Library folder.
local/publisher_registry.jsonJSON fileAutocomplete data for publisher/author names and tags across all libraries.
local/audit_trail.jsonJSON fileAppend-only audit log recording all lifecycle events (imports, deletes, rollbacks, package creations, integrity checks) with timestamps, usernames, and environmental context.

local/exports/DirectoryDefault output directory for exported packages and archives.

Package Backup Structure

The local/packages/ directory organizes cached packages by library name:

local/packages/
  MyLibrary/
    MyLibrary_v1.0.0_20260215-093012.hxlibpkg
    MyLibrary_v1.1.0_20260301-141530.hxlibpkg
  AnotherLib/
    AnotherLib_v2.0.0_20260220-110045.hxlibpkg

Every time a library is imported (or a system library is backed up on first run), a copy of the .hxlibpkg file is stored here. This enables:

  • Version rollback — revert to any previously installed version
  • Integrity repair — restore corrupted library files from the cached package
  • Audit trail — every version that was ever installed is retained as evidence

Settings Persistence

All application settings are stored in local/settings.json as a single JSON record. Settings are globally applied — any change made by one user is immediately visible to all other users on the same PC. Key settings include:

Night Mode (Dark Theme)

Library Manager supports a night mode (dark theme) that can be toggled in two ways:

  • Overflow menu — click the three-dot menu () and select Night Mode / Day Mode
  • Settings dialog — check or uncheck Enable night mode (dark theme) under the Display section

The dark mode preference is stored in local/settings.json as chk_darkMode and is persisted between application sessions. Because settings are global, the theme preference applies to all users on the PC.

Migration from Previous Versions

When upgrading from an earlier version of Library Manager, the application automatically migrates data from legacy storage locations:

Legacy LocationVersionMigration Action
<app>/db/Pre-1.3Settings and library data are merged into local/
HAMILTON\Library\LibraryManagerForVenus6\1.2–1.3Library registry, groups, tree, links, and audit trail are copied to local/
HAMILTON\Library\.LibraryManagerForVenus6\1.0–1.1Same as above (hidden folder variant)
HAMILTON\Library\LibraryPackages\1.2–1.3Cached package backups are copied to local/packages/

Migration is non-destructive — data is only copied if the destination file is empty or contains default seed data. Original files in legacy locations are not deleted.

Backup Recommendations

  • Back up the entire local/ directory regularly. It contains all application state and is the single source of truth.
  • The local/packages/ subdirectory can become large over time as package versions accumulate. Consider archiving or removing old versions periodically if disk space is a concern.
  • The local/audit_trail.json file is an append-only log that automatically rotates when it reaches 10,000 entries. Older entries are archived to timestamped backup files (e.g., audit_trail_2026-01-15T10-30-00-000Z.json). The active log file always contains the most recent entries.

Folder Permissions

The local/ directory must be writable by all users who run the application. The installer grants full control to the Users group on this directory. If you encounter permission errors, verify that the folder ACLs allow read/write access for the relevant user accounts or groups.

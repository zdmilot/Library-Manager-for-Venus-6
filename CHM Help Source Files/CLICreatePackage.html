CLI: create-package
Venus Library Manager » CLI Reference » create-package

Description

Creates a .hxlibpkg from library source files and metadata defined in a JSON specification file. This is the CLI equivalent of the GUI Library Packager, ideal for automated builds and CI/CD pipelines.

Usage

node cli.js create-package --spec <spec.json> --output <output.hxlibpkg>

Options

FlagRequiredDescription
--spec <path>RequiredPath to the JSON specification file (see Package Spec JSON Schema)
--output <path>RequiredOutput .hxlibpkg file path

Specification File

The spec file is a JSON document that defines all metadata and which files to bundle. It is validated against cli-schema.json. All file paths are resolved relative to the directory containing the spec file.

Required Fields

FieldTypeDescription
authorstringLibrary author name
versionstringLibrary version string (e.g., "1.0.0")
library_filesstring[]Paths to library payload files (at least one required)

Optional Fields

FieldTypeDescription
library_namestringOverride library name. Auto-detected from .hsl/.hs_/.smt if omitted
organizationstringAuthor's organization
venus_compatibilitystringVENUS version compatibility
descriptionstringHuman-readable description
tagsstring[]Searchable keyword tags
library_imagestringPath to icon/image file. Falls back to matching .bmp if omitted
demo_method_filesstring[]Paths to demo/example method files
help_filesstring[]Basenames of .chm help files
com_register_dllsstring[]Basenames of DLLs requiring COM registration

Library Name Auto-Detection

If library_name is not specified in the spec, it is auto-detected using this priority:

  1. Basename of the first .hsl file (without extension)
  2. Basename of the first .hs_ file (without extension)
  3. Basename of the first .smt file (without extension)
  4. Parent directory name of the first library file

Image Handling

If library_image is specified and the file exists, it is embedded as the library icon (base64-encoded in the manifest and packed into the icon/ directory).

If no image is specified, the packager looks for a .bmp file among the library files whose basename (lowercase) matches the library name. If found, it is used as the icon.

Examples

:: Basic package creation
node cli.js create-package --spec MyLib.spec.json --output MyLib.hxlibpkg

:: Output to a subdirectory
node cli.js create-package --spec specs/proj.json --output dist/proj.hxlibpkg

Validation

The command validates before building:

  • author is present
  • version is present
  • library_files has at least one entry
  • All referenced files exist on disk
  • All referenced help files exist on disk

If any validation fails, the command prints a descriptive error and exits.

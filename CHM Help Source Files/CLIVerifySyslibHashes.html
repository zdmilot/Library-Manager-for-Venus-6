CLI: verify-syslib-hashes
Library Manager » CLI Reference » verify-syslib-hashes

Description

Verifies the integrity of Hamilton system libraries by comparing the current metadata footer values against a previously generated baseline. Reports files that are tampered, missing, or errored.

Usage

node cli.js verify-syslib-hashes [--hash-file <path>] [--lib-dir <path>] [--json]

Options

FlagRequiredDescription
--hash-file <path>OptionalPath to the baseline file. Defaults to db/system_library_hashes.json
--lib-dir <path>OptionalLibrary directory to verify. Falls back to the database lib-folder link, then to the default HAMILTON Library path
--json, -jOptionalOutput results as structured JSON
--db-path <path>OptionalOverride user data directory (for database resolution)

Detection Criteria

For each baselined file, the verifier checks the current metadata footer against the stored values:

CheckConditionReported As
File missingFile does not exist on diskMissing
Footer removedFile exists but no longer has a metadata footerTampered - "Metadata footer removed"
Valid flag changedBaseline had valid=1 but current is valid=0Tampered - "Valid flag changed (1->0)"
Checksum changedCurrent $$checksum$$ differs from baselineTampered - "Checksum changed"
Read errorFile could not be parsedError
All checks passFooter present, valid flag and checksum matchOK

JSON Output

node cli.js verify-syslib-hashes --json

Returns a JSON object with four arrays: ok, tampered, missing, errors.

{
  "ok": [
    { "library": "HSLArrLib", "file": "HSLArrLib.hsl" }
  ],
  "tampered": [
    {
      "library": "HSLAppsLib",
      "file": "HSLAppsLib.hsl",
      "reason": "Checksum changed",
      "expected": "checksum=abc123",
      "actual": "checksum=def456"
    }
  ],
  "missing": [
    { "library": "HSLDevLib", "file": "HSLDevLib.hsl" }
  ],
  "errors": []
}

Exit Codes

CodeMeaning
0All files verified OK - no tampered, missing, or errored files
1Integrity issues detected (tampered, missing, or errors)

This makes the command suitable for CI/CD pipelines and automated health checks.

Library Directory Resolution

The library directory to verify is resolved in this priority order:

  1. --lib-dir command-line argument
  2. The lib-folder link from the Library Manager database
  3. Default: C:\Program Files (x86)\HAMILTON\Library

Examples

:: Basic verification with defaults
node cli.js verify-syslib-hashes

:: Verify a specific library folder
node cli.js verify-syslib-hashes --lib-dir "D:\Venus\Library"

:: JSON output for scripting
node cli.js verify-syslib-hashes --json

:: Use a specific baseline file
node cli.js verify-syslib-hashes --hash-file "backups/baseline_2024.json"

Error Conditions

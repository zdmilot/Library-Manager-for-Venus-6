Package Spec JSON Schema
Library Manager for Venus 6 » Package Spec JSON Schema

Overview

The package specification file is a JSON document that describes the metadata and file contents of a .hxlibpkg library package. It is used by the create-package CLI command to build packages. The spec file is validated against cli-schema.json (JSON Schema Draft-07).

Path Resolution: All file paths inside the spec are resolved relative to the directory containing the spec file.

Required Fields

FieldTypeConstraintsDescription
author string minLength: 3, maxLength: 29 Full name of the library author.
version string minLength: 1 Library version string. Semantic versioning is recommended but not enforced (e.g., "1.0.0", "2.3.1").
library_files string[] minItems: 1 Paths to library payload files (.hsl, .hs_, .hsi, .smt, .stp, .dll, .bmp, etc.). At least one file is required.
venus_compatibility string minLength: 1 VENUS version compatibility description (e.g., "4.7+", "5.0 - 6.x").
description string minLength: 1 Human-readable description of the library.

Optional Fields

FieldTypeConstraintsDescription
library_name string minLength: 1; no path separators Override library name. If omitted, auto-detected from the first .hsl, .hs_, or .smt file.
organization string - Author's organization or company.
github_url string format: uri URL to the library's GitHub repository. Must point to a valid /{owner}/{repo} path. See GitHub Repository Link for full validation rules.
tags string[] uniqueItems: true Searchable keyword tags. Over 60 tag keywords are reserved (case-insensitive) and must not be used — see the Reserved Tag Keywords section below for the complete list.
library_image string - Path to an icon/image file (.png, .jpg, .bmp, .svg, .gif, .ico). Falls back to matching .bmp if omitted.
demo_method_files string[] - Paths to demo/example method files installed under the Methods directory.
help_files string[] pattern: \.chm$ Basenames of .chm help files. Auto-detected from library_files during import if omitted.
com_register_dlls string[] pattern: \.dll$ Basenames of DLL files requiring 32-bit COM registration via RegAsm.exe /codebase. Must match entries in library_files. Important: VENUS is a 32-bit application — only the 32-bit RegAsm.exe from C:\Windows\Microsoft.NET\Framework\ must be used (never Framework64). See COM DLL Registration.

Additional Properties

The schema specifies "additionalProperties": true. Any unrecognized properties in the spec file are silently accepted and preserved in the generated package manifest. This enables forward compatibility — spec files created for future application versions (with new optional fields) can still be processed by older versions without validation errors. The unknown fields are carried through into the manifest.json inside the output package.

GUI behavior: If a user enters a restricted tag in the Library Packager or the Unsigned Library editor, an error is shown and the tag is blocked.

CLI behavior: If the tags array in a package spec file contains a reserved keyword, the create-package command will exit with a validation error.

Minimal Example

{
  "author": "Jane Smith",
  "version": "1.0.0",
  "library_files": [
    "MyLibrary.hsl",
    "MyLibrary.hs_"
  ]
}

In this case, library_name will be auto-detected as "MyLibrary" from the first .hsl file.

Full Example

{
  "library_name": "MyVenusLibrary",
  "author": "Jane Smith",
  "organization": "Acme Pharma",
  "version": "1.0.0",
  "venus_compatibility": "4.7+",
  "description": "My example Hamilton VENUS library.",
  "github_url": "https://github.com/acme-pharma/venus-library",
  "tags": ["example", "venus"],
  "library_image": "MyVenusLibrary.png",
  "library_files": [
    "MyVenusLibrary.hsl",
    "MyVenusLibrary.hs_",
    "MyVenusLibrary.hsi",
    "MyVenusLibrary.bmp"
  ],
  "demo_method_files": [
    "demo/Demo_MyVenusLibrary.hsl"
  ],
  "help_files": [
    "MyVenusLibrary.chm"
  ],
  "com_register_dlls": []
}

Auto-Generated Manifest Fields: The following fields are automatically added to the generated manifest.json during package creation and are not part of the spec file schema: format_version, app_version, windows_version, venus_version, package_lineage, and created_date. These are stamped by the application at build time. See Package Formats for the complete manifest schema.

Library Name Auto-Detection Priority

When library_name is omitted:

  1. Basename (without extension) of the first .hsl file in library_files
  2. Basename of the first .hs_ file
  3. Basename of the first .smt file
  4. Parent directory name of the first file in library_files

Image Handling

If library_image is specified and the file exists:

  • The image is base64-encoded and stored in the manifest as library_image_base64
  • The MIME type is auto-detected from the file extension
  • The image is placed in the icon/ directory inside the package

Supported MIME types:

ExtensionMIME Type
.pngimage/png

.bmpimage/bmp

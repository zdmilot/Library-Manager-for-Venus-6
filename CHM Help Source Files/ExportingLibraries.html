Exporting Libraries
Venus Library Manager » Exporting Libraries

Overview

Venus Library Manager can export installed libraries back into portable package files. Exports always reflect the current on-disk state of the library files (mutable export), so any changes made since the original import are captured in the exported package.

Single Library Export (.hxlibpkg)

GUI Workflow

  1. Click on a library card to open the Library Detail Modal
  2. Click the Export button
  3. Choose a save location — the suggested filename is <LibraryName>.hxlibpkg
  4. Venus Library Manager verifies that all expected library files exist on disk before exporting
  5. A new .hxlibpkg is built from the current files and metadata

What Is Included in the Export

The exported package contains:

  • manifest.json with current metadata (a new created_date is stamped)
  • All library files in the library/ directory (including .chm help files)
  • All demo method files in the demo_methods/ directory
  • Metadata fields: tags, base64-encoded image, COM DLL list

Restrictions

  • System libraries (Hamilton built-in) cannot be exported. They are read-only.
  • Deleted libraries cannot be exported.
  • If any expected library file is missing on disk, the export is aborted with an error message.

Multi-Library Archive Export (.hxlibarch)

GUI Workflow

  1. Open the Archive Export modal
  2. All non-deleted installed libraries are listed with checkboxes
  3. Select libraries individually, or use Select All / Select None
  4. Choose an output location for the .hxlibarch file
  5. Each selected library is re-packed from its current installed state

Archive Contents

The resulting .hxlibarch file contains:

  • An archive_manifest.json with archive metadata, library count, and included library names
  • One .hxlibpkg entry per selected library, each with its own manifest.json and payload

CLI Export

See CLI: export-lib for single exports and CLI: export-archive for archive exports.

Mutable Export

Note: Exports read files from their current location on disk, not from a cached snapshot. If library files have been modified since they were installed (for example, during development or customization), the export will include the modified versions. This is by design, supporting iterative development workflows.

If you need the original imported version, use Version Rollback to access the cached package from the package store.
